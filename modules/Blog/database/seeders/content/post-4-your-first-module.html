One of Saucebase's best ideas is the module scaffolding system. Instead of copy-pasting from a previous project or starting from a blank file, you run a single Artisan command and get a fully structured module with everything in the right place.

php artisan saucebase:recipe ModuleName

That's it. In seconds you have a module with a service provider, a controller, routes, Vue pages, migrations, a factory, a seeder, tests, and a Vite config — all following Saucebase conventions.

Enabling your module

After scaffolding, three more commands and your module is live:

composer dump-autoload
php artisan module:enable ModuleName
php artisan module:migrate ModuleName --seed

Then rebuild assets with npm run build or restart npm run dev.

What you get for free

  • Service provider registered and booted automatically
  • Routes loaded and scoped to the module
  • Filament plugin ready for your admin resources
  • Vue pages discoverable by Inertia out of the box
  • TypeScript types generated via php artisan module:generate-types
  • Playwright config auto-discovered by module-loader.js

Built for teams

Modules are self-contained by design. One developer owns the billing module, another works on the blog, a third builds a new feature module. There's no shared global state to step on, no merge conflicts in a monolithic service class.

The recipe system isn't just scaffolding — it's a statement about how features should be built. Modular, self-contained, copy-and-own. Add your Saucebase, make it yours.
